# 01 Fundamentos de Blockchain

## Objetivo

Entender qué es blockchain, cómo funciona a nivel general y por qué esta tecnología es importante en proyectos reales.

## ¿Qué es blockchain?

Blockchain es un registro digital compartido donde la información se guarda en bloques conectados entre sí.

Cada bloque contiene datos y una referencia al bloque anterior.
Eso hace que la información quede encadenada y sea más difícil de alterar.

## Idea simple

Puedes imaginar una blockchain como un cuaderno compartido entre muchas personas.

Cada vez que alguien escribe una nueva página:
- todos ven el cambio
- todos comparan que sea válido
- y la nueva página queda conectada con la anterior

Así, nadie controla el cuaderno por completo y cambiar páginas viejas no es fácil.

## Conceptos base

### Bloque

Un bloque es un paquete de información.

Puede contener:
- transacciones
- marca de tiempo
- referencia al bloque anterior
- otros datos técnicos de la red

### Cadena

La cadena es la unión de bloques en orden.

Cada bloque apunta al anterior.
Por eso se llama blockchain.

### Transacción

Una transacción es una acción registrada en la red.

Ejemplos:
- enviar criptomonedas
- interactuar con un smart contract
- guardar una operación en la blockchain

### Nodo

Un nodo es una computadora que participa en la red blockchain.

Dependiendo del tipo de nodo, puede:
- guardar una copia de la cadena
- validar información
- retransmitir datos a otros nodos

### Descentralización

Descentralización significa que el sistema no depende de un solo servidor o una sola empresa para funcionar.

En lugar de eso, varias computadoras participan y comparten el estado de la red.

## ¿Por qué importa blockchain?

Blockchain importa porque permite registrar información y ejecutar operaciones en sistemas donde no hace falta confiar en una sola entidad central.

Eso se usa en:
- pagos
- finanzas descentralizadas
- tokens
- trazabilidad
- identidad digital
- videojuegos Web3
- smart contracts

## ¿Qué problema intenta resolver?

Antes de blockchain, muchos sistemas digitales dependían de una autoridad central.

Ejemplos:
- un banco
- una empresa
- una plataforma
- un servidor principal

Blockchain intenta resolver el problema de coordinar valor e información entre muchas partes sin depender totalmente de un solo intermediario.

## Qué blockchain no significa

Blockchain no significa automáticamente:
- seguridad perfecta
- anonimato total
- velocidad alta
- bajo costo
- descentralización real en todos los casos

Eso depende de la red, del diseño y del uso que se le dé.

## Ejemplo del mundo real

Si haces una transferencia en un banco tradicional:
- el banco registra la operación
- el banco controla la base de datos
- el banco decide el estado final

Si haces una transacción en una blockchain pública:
- la red valida la operación
- varios nodos comparten la información
- el estado queda registrado en la cadena

## Resumen corto

Blockchain es:
- una estructura de datos
- una red distribuida
- una forma de registrar operaciones
- una base para aplicaciones descentralizadas

## Errores comunes

### Error 1
Pensar que blockchain es solo Bitcoin.

Bitcoin fue la primera gran aplicación, pero blockchain se usa en muchos otros contextos.

### Error 2
Pensar que blockchain reemplaza todo.

No todos los problemas necesitan blockchain.
En muchos casos una base de datos tradicional sigue siendo mejor.

### Error 3
Pensar que descentralización significa ausencia total de reglas.

Las blockchains también tienen reglas, validaciones y restricciones.
